VERBINDLICHE PROJEKTREGEL
- TVision wird aufgrund der Upstream-Strategie nicht angefasst.
- Standardnähe zu magiblot/TVision bleibt zwingend.
- Keine nicht standardnahen UI-Workarounds oder Overlay-Konstruktionen, wenn eine TVision-nahe Lösung möglich ist.
- Priorität: technische Exzellenz, saubere Architektur, robuste Implementierung. Nicht Geschwindigkeit.

AKTUELLER ARBEITSFOKUS
- Filename Extensions / Profiles / DEFAULT-Dialog (FE-Dialog) bleibt Referenzdialog für künftige Dialoge.
- Der große Nicht-TVision-C++20-/Refactoring-Zug ist inhaltlich abgeschlossen.
- Jetzt geht es primär um saubere Setup-Modellierung und ruhige Dialoggeometrie, nicht um sofortige Laufzeitfunktion.

WICHTIGE FACHLICHE ENTSCHEIDUNG
- Setup zuerst vollständig modellieren.
- Noch keine vorschnelle funktionale Implementierung der neuen Optionen.
- Funktionale Aktivierung später modulweise, erst nach Ausbau zentraler Komponenten:
  - Tree-Sitter
  - Code Folding
  - weiterer Ausbau von mrmac
- Die Dialoge dienen aktuell als Konfigurationsmodell / Architekturhilfe.

FE-DIALOG: AKTUELL VERBINDLICHE DESIGNREGELN
1. Profilreinheit
- Im FE-Dialog stehen nur Dinge, die sich sinnvoll auf File-Extension-Profile beziehen.
- Globale Themen wie Backups / Autosave gehören NICHT in den FE-Dialog.
- Dafür soll separat ein eigener Dialog aus IAS entstehen.
- Aktueller IAS-Eintrag „Backups / Temp / Autosave“ öffnet derzeit nur einen Dummy/Placeholder. Das ist vorübergehend akzeptiert, muss später durch den echten Dialog ersetzt werden.

2. Browse-/Picker-Regel (neuer Standard)
- Immer Lupe, nie "...".
- Die Lupe sitzt bündig am rechten Ende des zugehörigen Feldes.
- Dadurch ist visuell sofort klar, welcher Picker zu welchem Feld gehört.
- Diese Regel künftig auch im Passes-Dialog übernehmen.
- Datei-/URI-/Macro-Felder: Datei-Picker zulassen.
- Pfadfelder: Verzeichnis-/Pfad-Picker, keine Dateiauswahl.

3. Obere Geometrie des FE-Dialogs
- Links: Profiles-Listbox.
- Rechts daneben: ein zusammenhängender oberer Hauptcluster aller Text-/Numerikfelder.
- Wenn rechts neue Felder hinzukommen, wird die Profiles-Listbox einfach höher.
- New / Copy / Delete bleiben strukturell unverändert und wandern durch die größere Listbox nach unten.
- Die Button-Zeile soll auf Höhe des letzten Feldes des oberen rechten Clusters liegen.
- Danach zwei Leerzeilen Abstand.
- Erst darunter beginnt der untere Bereich mit Options und Radio-Clusters.

4. Untere Geometrie des FE-Dialogs
- Links unten: Options-Block (Checkboxen). Dieser darf künftig wachsen.
- Rechts daneben: Radio-Button-Cluster, ordentlich gruppiert, Breite pragmatisch passend.
- Keine kreuz-und-quer-Verteilung.
- Tab-Reihenfolge immer bewusst entlang der sichtbaren Geometrie halten.

5. Numerik-Regel
- Tab size bleibt MRNumericSlider.
- Right margin ist NICHT Slider, sondern klassisches numerisches Eingabefeld.

SETUP-INHALT: WAS BEREITS GILT
- Colors: erledigt.
- Language type: später zusammen mit Tree-Sitter / Syntax Highlighting.
- Compiler-Setup: viel später.
- Save editor status / ähnliche Status-Mitspeicher-Themen: bewusst nicht.
- Network file locking: immer an, keine Setup-Option.
- Edit mode „document“: wird weggelassen.
