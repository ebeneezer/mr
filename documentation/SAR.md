Der Match wird farblich highlighted, die Überschrift der Treffer Listbox ist die gesamte URL des Files. Klick/Enter auf einem Treffer bringt den User in den zugehörigen Editor mit diesem File andie Trefferstelle oder läd das File und stellt den Cursor auf den Treffer. Solange nichts neues gesucht wird behält die Listbox ihren Inhalt, so dass man den Multifile Search Dialog erneut rufen kann und zur nächsten Datei wechseln kann siehe documentation/pngsjpegs/Bildschirmfoto_20260420_185433.png Funktion "List files from last search". Die Option "Search files in memory" durchsucht zusätzlich zum "Starting path" auch alle bereits geladenene Files. Im Feld "Filespec" können mehrere glob Filenamen angegeben werden: Etwa "*.c *.cpp *.h *.hpp" durch whitespace getrennt, die im Searchpath oder rekursiv ab Searchpath durchsucht werden. Hier ist eine History Funktion wie bei den Pfaden/Filenamen einzubauen. Dasselbe beim Search Path - dort eine Path History einbauen.
