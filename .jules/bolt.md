**Learning:** Found a specific bottleneck where `piecewiseLineIndex` iterated character-by-character manually instead of deferring to the highly optimized `piecewiseCountLineBreaksInRange`, which uses SSE2 SIMD instructions.
**Action:** When working on text processing functions, always check if there's an existing vectorized operation that can be reused instead of implementing loops manually.
## 2024-10-25 - [Optimize find_first_of for small character sets]
**Learning:** Found a specific bottleneck in INI syntax highlighting where `find_first_of("=:")` was used. In the project's default C++ environment (GCC/libstdc++), calling `find(char)` multiple times and combining the results with `std::min` is significantly faster than a single `find_first_of` call due to `find` utilizing fast `memchr` and SIMD optimizations, while `find_first_of` often iterates characters. Attempted to optimize a UI dialog validation earlier, but realized that optimizing a UI cold path provides no measurable performance benefit and violates the strict constraint against premature optimization.
**Action:** When working on text processing or syntax highlighting hot paths, prefer multiple `find(char)` calls over `find_first_of` for small search sets. Avoid micro-optimizing UI cold paths or initialization logic where there is no measurable impact. Always add explanatory comments to optimizations.
